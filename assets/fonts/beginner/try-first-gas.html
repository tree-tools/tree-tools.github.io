Google Apps Script（GAS）を動かしてみる

このページでは、GASを使ったスプレッドシート作業の自動化を
初心者の方にも分かりやすく解説します。

※ GASの開発はPCでの作業が前提です。スマホでは編集や実行は困難なため、必ずPCで操作してください。

GASでできること

GASを使うと、普段の作業を自動化し、時間や手間を大きく削減できます。

  • スプレッドシート内のデータをチェックして自動修正
  • 毎週の報告メールを自動で送信
  • 複数シート間のデータを一括転記
  • PDF作成や定期処理の自動化

こうした作業を無料でできるのがGASの大きな魅力です。

GASを開く方法

ここではスプレッドシートからGASを開く手順を説明します。

  1. Google スプレッドシートを開く
  2. メニュー「拡張機能」→「Apps Script」を選択

サンプルコードを動かしてみる

実際にコードを書いてGASの動きを体験してみましょう。

①スクリプトファイルを開く

Apps Script エディタが開くと コード.gs というファイルが表示されます。
ここにコードを書きます。

②サンプルコードを入力

以下のコードをコピーし、エディタに貼り付けてください。

function myFirstFunction() {
  Logger.log("Hello, GAS!");
}

コードの意味

  • function は関数を定義するキーワードです。
  • Logger.log() はログ出力用の命令で、()内に書かれた内容をログに表示します。
  • 関数名は自由に付けられます。

③保存

エディタは自動保存されますが、ディスクアイコンを押して手動保存も可能です。

④実行

ツールバーの ▶ 実行ボタンを押して、スクリプトを実行しましょう。

初回実行時にはGoogleアカウントでの権限承認が必要です。

権限の承認方法はこちら

②～④の操作箇所は下の画像を参考にしてください。

スクリプト実行後

実行開始をしたらログ画面が開かれ、「お知らせ 実行開始」とログが表示されます。
処理が完了したら、「お知らせ 実行完了」と表示されます。

ログ画面は、「実行ログ」ボタンを押しても確認できます。

今回のスクリプトが正常に処理されれば、実行開始と実行完了の間に「情報 Hello, GAS!」と表示されます

スクリプト実行が失敗した場合

コードのタイプミスや不明な処理などがある場合は、実行ログにエラーとその理由が表示されます。

下の画像のエラーはコード.gsの2行目に書かれているLogger という関数は存在しないと記載されてます。

実際に自身が書いたコードでエラーが出た場合は、エラー文をコピーして検索をすると解決策が見つかります。

代表的なGASの活用例

実際の業務で使える、GASの具体例を紹介します。

スプレッドシートに書き込む

このコードを実行するとA1セルに「Hello, Spreadsheet!」と貼り付けます。

function writeToSheet() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  sheet.getRange("A1").setValue("Hello, Spreadsheet!");
}
